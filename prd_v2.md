# AI ID Photo Studio - V2 (Custom Reference Feature) PRD

## 1. 개요 (Overview)
기존 V1 버전은 사전 정의된 17종의 의상/헤어 프리셋(Presets)을 단순히 클릭하여 적용하는 제한적 방식이었습니다. 
새롭게 기획된 **V2 버전**은 **사용자 맞춤형 레퍼런스 이미지 적용(Custom Reference Image)** 기능을 핵심으로 합니다. 사용자가 직접 입히고 싶은 의상 사진을 추가로 업로드하면, 고객의 핏과 체형에 맞춰 해당 의상을 자연스럽게 합성해내는 전문가 수준의 합성 도구를 목표로 합니다.

## 2. 유저 플로우 (User Flow)
1. **듀얼 업로드 초기 화면 (Step 1):**
   - 앱 첫 접속 시 화면 중앙에 **두 개의 업로드 영역(Dropzone)**이 나란히 표시됩니다.
   - **좌측 영역:** 고객(Client)의 얼굴이 포함된 원본 사진 업로드
   - **우측 영역:** 고객에게 입히고 싶은 의상/스타일을 담은 '레퍼런스(Reference) 사진' 업로드
2. **AI 합성 실행 (Step 2):**
   - 두 장의 사진이 모두 업로드된 상태에서 활성화되는 '변환 실행(Generate)' 버튼을 누르면 API 전송이 시작됩니다.
   - 원본 사진과 레퍼런스 사진, 그리고 특수한 제어 프롬프트가 함께 Gemini 멀티모달 규격으로 전송됩니다.
3. **최종 결과물 비교 화면 (Step 3):**
   - 로딩 완료 후, V1의 직관적인 좌-우 분할 레이아웃으로 진입합니다.
   - **좌측 블록:** 고객의 원본 사진 고정 배치
   - **우측 블록:** 머리(얼굴)는 원본을 100% 유지한 채 체형 부위에만 레퍼런스 의상이 완벽하게 핏(Fit)되어 렌더링된 최종 결과물 출력

## 3. 핵심 요구사항 (Core Requirements)
* **프론트엔드 레이아웃 개편:**
  - V1의 기존 소스 코드는 보존하면서, 사용자 경험을 해치지 않는 선에서 유연한 2단 레이아웃을 구성합니다.
  - 레퍼런스 이미지를 업로드하고 미리보기(Preview)할 수 있는 상태 관리(`referenceImage` state)가 추가로 필요합니다.
* **멀티모달 API 페이로드 전송:**
  - API 요청 시 전달되는 페이로드(`parts` 배열)에 **이미지가 2장(원본 사진 + 레퍼런스 사진)** 들어가도록 `geminiService.ts`의 로직 범용성을 확장해야 합니다.
  - 레퍼런스 이미지를 지칭하는 프롬프트 내부 명칭(예: "the second attached image")을 명확히 시스템 스펙에 맞게 바인딩합니다.
* **초정밀 제어 프롬프팅 적용:**
  - 고객이 제시한 초정밀 8K DSLR 스펙 프롬프트를 시스템 내부 프롬프트로 완전히 내재화합니다.
  - **주요 제어 항목:**
    1. **Identity preservation (최우선):** 얼굴 포즈, 각도, 이목구비 100% 동일 유지. 왜곡/나이 변화 절대 금지.
    2. **Outfit (의상):** 두 번째 첨부된 레퍼런스 이미지를 바탕으로 의상만 교체.
    3. **Background (배경):** **원본 사진의 배경을 100% 동일하게 유지**. 어떠한 배경 변형이나 임의 디자인도 금지. (Background exact preservation)
    4. **Lighting & Quality (조명 및 품질):** 부드러운 3점 조명, 얼굴 균등 노출, 8K DSLR 리얼리즘, 과샤픈 금지.
